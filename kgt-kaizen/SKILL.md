## 2. PDCA サイクル

```
Plan（計画）:
  1. 現状把握: データで現状を数値化
  2. 目標設定: SMART（具体的・測定可能・達成可能・関連性・期限）
  3. 原因分析: なぜなぜ5回・特性要因図（フィッシュボーン）
  4. 改善計画: 実施項目・担当者・期限の明確化

Do（実施）:
  5. 小さく試す（パイロット）
  6. 計画通り実施・記録

Check（確認）:
  7. 改善前後の数値比較
  8. 目標達成度の評価
  9. 予期しない副作用の確認

Act（改善・標準化）:
  10. 効果あり → 標準化（作業手順書・システム化）
  11. 効果なし → 次の Plan へ（別の原因仮説を検討）
  12. 水平展開（他工程・他ラインへの適用）
```

## 3. 5S

```
整理（Seiri）:
  「必要なもの」と「不要なもの」を分ける
  → 赤札作戦: 使用頻度で判定・不要品を廃棄
  → 基準: 直近1ヶ月使わなかったものは移動・処分検討

整頓（Seiton）:
  「必要なものを、必要な時に、すぐ取り出せる」配置
  → 定位置管理: 置き場所と数量を固定・表示
  → ツールシャドウボード: 工具の定位置表示

清掃（Seiso）:
  「汚さない・汚れたら即清掃」
  → 点検をかねた清掃（設備の異常を発見）
  → 清掃責任区域の明確化

清潔（Seiketsu）:
  3S（整理・整頓・清掃）の維持・標準化
  → 写真・基準書による「あるべき姿」の見える化
  → 定期パトロール・チェックシート

躾（Shitsuke）:
  ルールを守る習慣・組織文化
  → OJT・定期教育・目標共有
  → リーダーによる模範行動
```

## 5. なぜなぜ分析（5 Whys）

```
目的: 問題の表面的な原因ではなく根本原因を特定する

手順:
  問題を明確に定義する
    ↓ なぜ1: なぜその問題が起きたか？
    ↓ なぜ2: なぜなぜ1の原因が起きたか？
    ↓ なぜ3: ...
    ↓ なぜ4: ...
    ↓ なぜ5: 根本原因（管理・仕組みのレベル）

例:
  問題: 顧客への納品が3日遅れた
  なぜ1: 外注品の入荷が遅れた
  なぜ2: 外注先の製造が遅延した
  なぜ3: 外注先への発注が遅かった
  なぜ4: 内製→外注切替の判断が遅れた
  なぜ5: 外注化判断のタイミング基準が未設定
  根本原因: 外注化を決断するための明確な基準・トリガーがない
  是正処置: 工程進捗が〇%以下になったら即外注化を検討するルールを策定

注意:
  "なぜ"が再現性のない個人の失敗に向かないよう注意
  → 「確認を怠った」ではなく「確認しなくても気づける仕組みがなかった」へ
```

## 6. OEE（設備総合効率）

```
OEE = 時間稼働率 × 性能稼働率 × 良品率

時間稼働率 = (稼働時間 - 停止ロス) / 稼働時間
  停止ロス: 故障・段取り・調整・材料待ち

性能稼働率 = (実際の生産数 × 基準サイクルタイム) / 稼働時間
  性能ロス: チョコ停・速度低下

良品率 = 良品数 / 生産数
  品質ロス: 不良・手直し

世界クラスの目標値:
  時間稼働率: 90% 以上
  性能稼働率: 95% 以上
  良品率:     99.9% 以上
  OEE:        85% 以上

6大ロス（OEE の構成要素）:
  1. 故障ロス（停止）
  2. 段取り・調整ロス（停止）
  3. チョコ停ロス（性能）
  4. 速度低下ロス（性能）
  5. 立上がりロス（品質）
  6. 不良・手直しロス（品質）
```

## 7. 標準作業（Standardized Work）

```
標準作業の3要素:
  1. タクトタイム（顧客要求レートに同期する速度）
  2. 作業順序（最も効率的な手順）
  3. 標準手持ち（作業を回すのに必要な最小の仕掛品数）

標準作業票:
  作業者が1サイクルで行う全動作を記述
  → タイムチャートで各動作の時間を可視化
  → タクトタイムとの差異をグラフ化（山積みチャート）

標準化の手順:
  1. 現状の作業を観察・計測（少なくとも10サイクル）
  2. 最速かつ再現性ある手順を標準に選ぶ
  3. 標準作業票に記録
  4. 全作業者に教育・訓練
  5. 定期的に見直し（改善があれば更新）
```

## 8. 改善テーマの優先付け

```python
def prioritize_kaizen(impact: dict) -> str:
    """
    改善テーマの優先度（影響 × 緊急度マトリクス）
    """
    score = (
        impact.get("defect_loss",       0) / 1000 +   # 不良損失（単位: バーツ）
        impact.get("overdue_orders",    0) * 5  +   # 納期遅延件数
        impact.get("customer_complaints", 0) * 20 +  # 顧客クレーム件数
        impact.get("downtime_hours",    0) * 2       # 設備停止時間
    )
    if score > 50:
        return "🔴 最優先（今週着手）"
    elif score > 20:
        return "🟡 次期（今月中）"
    return "🔵 継続監視"
```
