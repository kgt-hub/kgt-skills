# 部品設計専門エージェント（KGT グループ共通）

機械部品の設計から量産に至るまでの実践的な設計知識を提供します。

---

## 1. 部品設計の基本プロセス

```
1. 機能要件の明確化
   → 部品が果たすべき機能・性能・耐久性を定義
   → 使用環境（温度・湿度・化学物質・荷重）を明確化

2. 概念設計（コンセプト）
   → 機能を実現する構造・形状のアイデア出し
   → 複数案を比較（トレードオフ分析）

3. 詳細設計
   → 寸法・公差・材料・表面粗さの決定
   → 標準部品（ネジ・軸受・シール）の選定

4. DFM レビュー（製造性確認）
   → 加工できるか・コストは適切か

5. 設計検証
   → 計算・シミュレーション・試作品評価

6. 図面作成・発行
   → JIS/ISO 準拠の図面作成
   → 承認・版管理

7. 量産移行
   → PPAP / 初回品確認 → 量産
```

### 寸法連鎖（公差積み上げ）

```python
def worst_case_tolerance_analysis(tolerances: list[float]) -> dict:
    """
    最悪ケース解析（Worst Case Analysis）
    全部品の公差が最悪方向に重なった場合の組立誤差を計算

    tolerances: 各部品の公差（片側値）のリスト
    """
    total_tolerance = sum(tolerances)
    return {
        "method": "最悪ケース解析",
        "total_tolerance": total_tolerance,
        "note": f"全部品の公差が最悪方向に重なった場合 ±{total_tolerance:.4f}",
        "verdict": "必ず成立する（100%保証）",
        "cost_impact": "公差が厳しくなる → コスト増",
    }

def rss_tolerance_analysis(tolerances: list[float]) -> dict:
    """
    RSS（Root Sum Square / 統計的公差解析）
    統計的に各公差が独立ならば、2乗和の平方根で組立誤差を推定

    精度: ±3σで 99.73% の品質保証
    """
    import math
    total_tolerance = math.sqrt(sum(t**2 for t in tolerances))
    return {
        "method": "RSS（統計的公差解析）",
        "total_tolerance": round(total_tolerance, 4),
        "note": f"±3σ で 99.73% 成立確率 ±{total_tolerance:.4f}",
        "verdict": "量産時に適用（0.27% の不成立を許容できる場合）",
        "cost_impact": "最悪ケースより公差が緩くなる → コスト低減",
    }

# 例: 3部品の寸法連鎖
# 部品A ±0.05, 部品B ±0.03, 部品C ±0.04
tolerances = [0.05, 0.03, 0.04]
# 最悪ケース: ±0.12
# RSS: ±0.071
```

## 5. 応力・強度設計の基礎

### 基本応力計算

```python
import math

def tensile_stress(force_N: float, area_mm2: float) -> float:
    """引張応力（MPa = N/mm²）"""
    return force_N / area_mm2

def bending_stress(moment_Nmm: float, section_modulus_mm3: float) -> float:
    """曲げ応力（MPa）"""
    return moment_Nmm / section_modulus_mm3

def section_modulus_circle(diameter_mm: float) -> float:
    """円形断面の断面係数（mm³）"""
    return math.pi * diameter_mm**3 / 32

def safety_factor(allowable_stress_MPa: float, actual_stress_MPa: float) -> float:
    """安全率（目安: 動荷重 2.0〜3.0 / 静荷重 1.5〜2.0）"""
    return allowable_stress_MPa / actual_stress_MPa

# 例: φ10mm シャフト、曲げモーメント 5000 N·mm
# 断面係数 = π×10³/32 = 98.2 mm³
# 曲げ応力 = 5000 / 98.2 = 50.9 MPa
# S45C 降伏点 ≈ 370 MPa → 安全率 = 370 / 50.9 ≈ 7.3（十分な余裕）
```

```python
DESIGN_FMEA_TEMPLATE = {

def calculate_rpn(severity, occurrence, detection):
    """RPN = 深刻度 × 発生度 × 検出度"""
    rpn = severity * occurrence * detection
    if rpn >= 200:
        return rpn, "🔴 即時対応"
    elif rpn >= 100:
        return rpn, "🟡 対策検討"
    return rpn, "🔵 監視継続"
```

## 7. 部品コストの影響因子

```
コストドライバーと影響度（大→小）:

1. 材料費（30〜60%）
   → 材料グレードの最適化（オーバースペックを避ける）
   → 素材形状の選定（丸棒 vs 角棒 vs 板材）

2. 加工工数（20〜40%）
   → 工程数の削減（工程集約・複合加工）
   → 段取り時間（類似形状のまとめ加工）

3. 公差・精度（10〜30%）
   → 公差 ±0.1 → ±0.01mm でコスト2〜5倍増
   → 必要箇所にのみ厳格な公差を適用

4. 表面処理・熱処理（5〜20%）
   → 必要性を再評価（全面処理 vs 部分処理）

5. 検査費用（5〜15%）
   → 全数検査 vs 抜取検査
   → 管理図による工程内管理でコスト削減

コスト削減の基本アプローチ:
  □ 標準品・流用品の活用（設計新規 < 既存流用）
  □ 部品点数の削減（2部品 → 1部品に統合）
  □ 加工基準の統一（1チャックで全加工）
  □ 寸法公差の緩和（機能に影響しない部位）
  □ 量産化による単価低下（数量効果）
```
